Models Used

  • Flare probability (Forecasting) blends a McIntosh classification + Mount Wilson magnetic-class lookup table, NOAA's own issued per-region probability (shown for comparison), and a Poisson "hot region" term from the region's flares-in-the-last-24h count.
  • Nowcasting converts the live GOES long-channel X-ray flux directly into the same A/B/C/M/X classification NOAA uses for flare events, plus a rising/falling trend from the raw flux over the last 30 minutes — this is "what is the Sun doing right now," not a probability.
  • CME Earth-arrival time (CME Tracker) uses the analytic Drag-Based Model of Vršnak et al. (2013, Solar Physics 285) — the same physics behind NOAA/ESA's operational CME-arrival tools — solved for a low/best/high arrival window, and tracked as a percent-complete progress bar from elapsed time vs. estimated transit time.
  • Short-term trend forecasts (Kp, solar wind speed) are a simple linear least-squares extrapolation of the last few hours of data — a naive statistical projection, not a physics-based forecast. It will visibly diverge from reality whenever a CME or corotating interaction region actually changes conditions.
  • Aditya-L1 cross-validation downloads real SoLEXS Level-1 X-ray light curves via ISRO's PRADAN portal (ISSDC) and flags which GOES-detected flares show a matching count-rate enhancement, as independent confirmation from a second satellite in a different orbit.

Limitations

  • CME initial speed is estimated from flare GOES class (CDAW-catalog statistics), not measured by coronagraph — treat transit times as order-of-magnitude, not precise forecasts.
  • No model can predict the exact moment a region will flare.
  • Storm severity depends heavily on IMF Bz, which is only reliably knowable once a CME sheath reaches L1, roughly 30–60 minutes before Earth impact — a physical limit of the observing geometry, not a modeling gap.
  • The Kp/solar-wind "forecast" lines are naive linear extrapolations of recent history, included so you can see where things are headed if nothing changes — not a substitute for the physics-based CME arrival model.
  • Aditya-L1/SoLEXS cross-validation uses raw total count rate vs. the day's median as an activity proxy, not a calibrated flux-to-GOES-class conversion, and Level-1 data has a multi-day publication latency (not real-time).
  • NOAA's real-time solar wind feed can return HTTP 200 with data that's hours stale during spacecraft/ground-station gaps — this dashboard checks the payload's own timestamp and flags it as stale when that happens.
